## KGI（最終ゴール）

| # | KGI | 成功のイメージ |
|---|-----|----------------|
| **1** | **「あの政治ネタ、今どうなった？」の定番サイトになる** | 案件ページが検索・SNS で引用される |
| **2** | **平易語＋一次ソース（議事録・X スクショ）で信頼を取る** | 「難しい政治メディア」ではなく「分かる記録」 |
| **3** | **複数レーンで月 ¥10 万（ストレッチ ¥15 万）** | Ad + アフィ + Shorts + 月額 + スポンサ等 |

※ リポ仮称 `kokkai-voice`。本番ブランド確定後も KGI は不変。

---

## サイトの目的（前提）

**政治で関心が高いこと**（国会・都政・リコール・論争）を **案件単位** で追い、  
平易語・Good/Bad・コメントで **国民の関心と感情** が見えるメディア。

jin-log（体験記・腎）とは **別ブランド・別収益**。

# 公開まで（Pre-launch）

## Pre-launch KGI

| KGI | 達成条件 |
|-----|----------|
| **β 公開できる** | ドメイン・HTTPS・利用規約・最低限の案件が揃っている |
| **差別化が 1 秒で伝わる** | トップ or 案件 1 本で「今どうなった・平易語・X スクショ」が分かる |

## Pre-launch KPI

### A. プロダクト完成

| 指標 | 現状 | 公開までの目標 | 測り方 |
|------|------|----------------|--------|
| UI サンプル | v1 完了 | **本番 Astro に移植** | デプロイ URL |
| 案件ページ数 | 1（デモ） | **本番 5 案件以上**（国会2・都政/政治2・リコール等1） | 公開 URL カウント |
| タイムラインイベント | デモのみ | **案件あたり平均 4 イベント以上** | 目視 |
| Good/Bad | デモ JS | **😊/😠 本番 DB 保存** | D1 / 管理画面 |
| コメント | デモ | **投稿→モデレーション→公開** | フロー通過 |
| X スクショ | UI モック | **実スクショ 3 件以上** を本番掲載 | R2 画像 |
| 国会データ | 未 | **議事録 API 取得 PoC 1 本以上** | `scripts/` + 出典リンク |

### B. 信頼・法務

| 指標 | 目標 | 測り方 |
|------|------|--------|
| 利用規約・プライバシー | **公開前必須** | ページ URL |
| 免責（AI 要約・ユーザー投稿） | **記載済** | フッター / 規約 |
| 公選法・リコール注意 | **チェックリスト完了** | `docs/` or オーナー OK |
| モデレーション方針 | **文書化** | 内部 1 ページ |

### C. インフラ

| 指標 | 目標 | 測り方 |
|------|------|--------|
| ドメイン | **取得済** | オーナー |
| 本番デプロイ | **Cloudflare Pages** | HTTPS 200 |
| 計測 | **GSC + Cloudflare Analytics** | ダッシュボード |
| AdSense | **申請済**（通過は公開後 KPI） | 管理画面 |

### D. 集客準備（公開と同時）

| 指標 | 目標 | 測り方 |
|------|------|--------|
| X アカウント | **1 本開設**（jin-log と別） | @handle |
| 公開告知 | **公開時 1 本 ＋ 案件追加のたび** | X |
| OG 画像 | **案件ページ対応** | シェアプレビュー |

### Pre-launch 完了チェック（KGI 達成の Yes/No）

- [ ] **話題案件 1 本**が実データで公開可能
- [ ] 利用規約・プライバシー掲載
- [ ] 😊/😠 が DB に記録される
- [ ] X スクショ実例が 3 件以上
- [ ] GSC 登録・サイトマップ送信
- [ ] オーナーが **公開 OK** を出した
