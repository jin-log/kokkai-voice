# ライター編集方針 — 公言と実行のギャップ

最終更新: 2026-06-27  
**正本。** KVライターエージェントは本書＋`docs/policy-matrix.md`を必読。

---

## 基本姿勢（オーナー方針）

**公約が守られなくて当然、という空気を記事で助長しない。**

メディアや政府説明が「検討中」「慎重に議論」で埋め尽くす中で、  
**いつ・何と約束したか／いま何が起きているか** を消さず並べるのが本サイトの存在理由。

- 党首・政権の **約束は評価の起点**。国会の最新発言で上書きしない
- 「まだ検討」は事実だが、**未履行の事実とセット** で書く
- 読者に「当たり前の遅れ」と思わせない。**時計（経過日数・期限）** を必ず入れる
- 感情語（裏切り・見送り）はタイトルに入れない。**数字と引用** で読者自身が判断する

---

## サイトの芯

読者が知りたいのは **「言ったこと」と「やったこと」の差** である。

- スピード（いつやると言ったか vs いま何日／何カ月経ったか）
- 規模（ゼロと言ったか vs 5%にとどまったか）
- 状態（「すぐ実施」と言ったか vs 「検討中」のままか）

国民から見れば、ここがズレていると **裏切り** に感じる。  
ライターはその感情を代弁しない。**日付・数字・発言引用で並べる** のが仕事。

## タイトル — 国民の具体的疑問（2026-06-28 改定）

**サイトキャッチ**「あの話どうなった？」はブランド。**案件タイトルには `— あの話どうなった？` を付けない。**

| × 抽象ラベル（NG） | ○ 具体の疑問（OK・オーナー例） |
|-------------------|-------------------------------|
| 物価高対策 | 物価高で確保した1兆円予備費、何に使われた？ |
| 食料品の消費税 | 食料品消費税ゼロ公約、スーパーはまだ8%？ |
| 防衛費・安保 | 防衛費増で増税はある？安保三文書と財源 |
| `{政策名} — あの話どうなった？` | 出生率改善予算とその成果実績 |
| | 副首都構想って何？ |
| | 大阪都構想のメリットデメリット |

**5つの型**

1. **数字＋結果** — 予算・税率・人数と「いまどうなったか」
2. **とは？** — 制度・構想の定義（読者が名前だけ聞いた論点）
3. **メリットデメリット / 推移** — 賛否の整理表向き
4. **公約vs現状** — 約束と今のギャップ（数字付き）
5. **経過＋結末** — `告発から2年、結局どうなった？`

**タイトルに入れない:** 裏切り・見送り・嘘・逆行 等の評価語（法務）。野党の強い言葉は X 枠へ。

**フック（nowSummary 1行目・X）:** タイトルの疑問に対する**結論先出し**＋時計。

## 内容の心理（中立・事実）

読者の多くは「税金・政治への疲れ」の中で、**自分の疑問だけ**を解きたい。

| 読後感（目指す） | 書き方 |
|------------------|--------|
| 今そうなってるのね | いまの数字・状態を先に |
| そういう事だったのか | `plainExplanation` 1段落目でタイトルの疑問に答える |
| 公約と違うじゃん（自分で判断） | 公言と行動を並べ、代弁しない |

**plainExplanation 1段落目:** タイトルと同じ問いへの回答（定義 or 結論）。2段落目以降で経緯。

**禁止:** 政策名だけの抽象サマリ / 政府説明のポジティブ要約だけ / タイトルと無関係な国会論点

## nowSummary（いまの結論）— 3行の型

**1行目: 公約 vs 現状（時計）**

- 公約の文言（誰が・いつ・何と言ったか）
- いまの状態（実施済み / 未実施 / 法案未提出 など）
- 経過日数・月数（起点日を明示）

**2行目: 言ったスピード vs 実際のスピード**

- 「早期実施」「〇月までに」等の **本人・党の期限**
- その期限から何日／何カ月過ぎたか
- いま政府が言っている次の期限（国民会議・中間まとめ等）

**3行目: いま国会で何を言っているか（中立）**

- 「検討中」だけで終わらせない。**検討中である事実** と **未実施である事実** を並べる
- **国会議事録未掲載（Phase A）** のときは、報道・Xベースで書き「国会原文は掲載待ち」と明示する（サイトが自動表示）

---

### × 禁止パターン

```
政府は「食料品の消費税ゼロ」をつなぎ策として前向きに検討中
```

→ 政府説明の **ポジティブ要約だけ** 。公約・期限・未実施が消える。

### ○ 必須パターン

```
総選挙公約「食料品消費税ゼロ」は2026年6月時点で未実施。
高市総理は早期実施を繰り返す一方、国会で示された目標（2026年3月31日）から約3カ月経過。
政府は社会保障国民会議で「つなぎ措置として検討」と説明し、中間まとめを2026年夏前としている。
```

## arcSummary / timeline

### タイムライン最低本数（2026-06-28 必須）

| 種別 | 最低 | 内容 |
|------|------|------|
| **X** | **3件** | `timeline` に `type: "x_post"`。`xPosts` の url_found を同期 |
| **国会** | **3件** | `type: "speech"` + `kokkai.ndl.go.jp` URL（行政案件も国会論点を拾う） |
| **合計** | **6件以上** | milestone / source は補助。上記を満たしてから追加 |

**Phase A（国会待ち先行公開）:** JSON に `"dietPending": true` → **X3 + タイムライン3件** で公開可。国会は載ったら追記して `dietPending: false`。

チェック: `node scripts/check-case-page.mjs --slug {slug}` の E1/E2/E3。  
一括強化: `node scripts/enrich-timeline-all.mjs`（**追記・同期のみ。既存行の書き換えはしない**）

### タイムライン追記ルール（2026-06-30）

- 新イベントは `timeline[]` **末尾に追加**（`id` は一意）
- **既存行の文言は変えない**（誤字・法務・削除済み注記のみ例外）
- X スクショ再取得は同一 `id` の `xPost` 更新のみ可

### メリット・デメリット（2026-06-28 必須）

- フィールド: `prosCons.merits[]` / `prosCons.demerits[]`
- 各項目 **必須**: `text`, `figure`（公表数値）, `sourceUrl`, `sourceLabel`
- **政治家の主張そのものはメリットにしない** — 省庁統計・予算・法案の公表数値のみ
- **× メリット禁止**: APIヒット件数・言及件数・「国会でよく話題に」等（読者に意味がない）
- **デメリット**: 状態（未成立・遅れ）だけで終わらせない。**読者にとって何が困るか**を1文で書く
- 表示: **「経緯をざっくり」の直上**（PCは左右2カード）
- チェック: J1_prosCons

### 経緯 arcSummary（2026-06-30 必須）

- **日付ごとに内容が異なること**。同じ1文のコピペは掲載禁止
- 各日 = **その日に起きた出来事**を15〜40字で（法案提出・可決・答弁・反対表明 等）
- ×「論点が表面化」だけを6日分繰り返す

### 根拠 summaryBullets（2026-06-30 必須）

- **経緯のコピー禁止**。結論を支える**別角度の事実**（数値・誰が何と言ったか・法案名）
- 形式: `YYYY-MM-DD：主体— 具体的事実（国会議事録）。`
- ×「〜をめぐる国会論点（国会議事録）」のテンプレ連打

### 公言と行動 policy-matrix（2026-06-30 必須）

| 欄 | 書くこと | × 禁止 |
|----|----------|--------|
| **方針** | その会派の立場を第三者が1〜2文で要約 | 議事録の切り出し・「…」で終わる抜粋 |
| **行動** | **何をしたか**（提出・可決・質疑・答弁の内容を動詞で） | 「本会議（日付）での発言」だけ |
| **symbolReason** | 「方針Aに対し行動B（日付）」 | 自動生成の一言 |

経緯は **古い順ではなく新しい順** で、必ず含める:

1. **公約・党首発言**（日付・出典）
2. **期限・目標日の表明**（「いつまでに」の発言）
3. **期限経過**（事実。「〇日経過・税率変更なし」）
4. **現在の公式説明**（検討中・国民会議 等）

「検討している」は **4の事実** として書く。**1〜3を省略しない**。

## 停滞理由の書き方（2026-06-28 追記）

**公言して終わりは記載しない。** 公言してるから良し、が一番ダメ。

読者向け4列（nowSummary / arcSummary で必ず意識）:

| 列 | 内容 |
|----|------|
| 公言 | 誰が・いつ・何と言ったか |
| 期限 | 本人・党が示した目標時期 |
| 経過 | 起点から何日・何カ月 |
| その後 | 変わった事実。**なければ「変更なし」** |

**言い訳の代弁は禁止。** ただし国会・会見・報道の**公式説明**は事実として載せる。  
**なぜ進んでいないか**＝公式説明＋客観的事実（法案未提出等）＋報道（「〜と報じられている」）。

Skill 正本: `.cursor/skills/kokkai-writer/SKILL.md`

## policy-matrix（〇×）

`docs/policy-matrix.md` の **公約優先ルール** に従う。

- 自民に「公約なし」と書くのは **誤り**（総選挙公約に記載 — 国会で片山大臣が言及）
- `symbolReason` に **公約との具体的ギャップ** を必ず1文
  - 例: `公約「食料品消費税ゼロ」に対し、2026年6月時点で税率変更・法案提出なし。国民会議での結論待ち`

記号:

| 公言 | 行動 | 記号 |
|------|------|------|
| 0%・早期実施 | 未実施・検討継続 | **▲**（期限・規模ずれ） |
| 0% | 増税に賛成 | **❌** |

## 経過日数の算式

起点は **複数並べてよい**。必ず `arcSummary` に日付を残す。

| 起点 | 用途 |
|------|------|
| 公約掲示日（総選挙・党公約PDF） | 公約から〇カ月 |
| 総理就任日 | 政権発足から〇カ月 |
| 本人が言った実施目標日 | 期限から〇日経過 |

**ハルシネ禁止** — 日付は国会発言・公約PDF・党HPのみ。推測日は書かない。
