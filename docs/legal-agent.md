# 法務チェックエージェント（別チャット用）

**役割:** アップロード前のコンテンツスキャン。問題あり → CEOに報告。問題なし → スルー（報告不要）。

## デプロイ前必須

```bash
node scripts/check-publish-ready.mjs --slug bouka-taisaku
```

`publishReady: true` かつ `legalReview.status: "ok"` でないと exit 1。全件集計は `--all`。

## 使い方

1. Cursorで **新規チャット** を開く（タイトル: `kokkai-voice 法務`）
2. `@data/articles/` または特定 slug を添付
3. プロンプト例:

```
legal-check ルールに従い、添付記事をアップロード前チェック。
問題があるものだけ箇条書きで報告。問題なしは「問題なし」とだけ。
```

## チェック項目

| # | 論点 | 引っかかったら |
|---|------|----------------|
| L1 | **公職選挙法** — 選挙期間中の特定候補者への呼びかけ・投票誘導 | CEO報告 |
| L2 | **名誉毀損** — 事実と異なる断定、個人への人格攻撃 | CEO報告 |
| L3 | **引用の正当性** — 国会原文リンク・出典なしの発言引用 | CEO報告 |
| L4 | **個人特定** — 実名・住所・連絡先の不要な掲載 | CEO報告 |
| L5 | **AI表示** — 要約に「AI」「原文抜粋」ラベル欠如（将来AI適用時） | CEO報告 |
| L6 | **Xスクショ** — 改変画像、出典URLなし | CEO報告 |
| L7 | **PR・実名** — 運営者実名・WHOIS・特商法表記の漏洩 | CEO報告 |

## 現フェーズの注意

- 現在の記事は **国会API原文抜粋のみ**（AI要約未適用）→ L5は該当時のみ
- X枠は **pending** → スクショ追加時に再チェック
- **問題なしが大半なら報告しない** — ノイズ削減

## CEOへの報告フォーマット

```
【法務】{slug}
- 論点: L2 名誉毀損
- 箇所: ...
- 修正案: ...
```
