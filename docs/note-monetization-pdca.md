## 無料 vs 有料（線引き）

| 無料（サイト） | 有料（note メンバー） |
|----------------|----------------------|
| 案件ページ（公言・経過・〇×） | **週次ダイジェスト**（新規・更新案件まとめ） |
| プレリリース記事20本+ | **深掘り1本/週**（1案件を時系列で更新） |
| X @seiji1192site 更新告知 | **更新アラート先行**（24h早い） |
| 😊😐😠 リアクション | メンバー限定「公言vs行動」差分表（PDF/HTML） |

**鉄則:** サイトの核心（1案件の整理表）は無料のまま。**「追いかけ続ける便利さ」** を売る。

## 「パッとあげたい」と思わせるUX（デザイナー案）

### 配置（優先順）

1. **記事読了直前** — `SupportCta`（いまの感情が残っている）
2. **記事冒頭** — `ShareBar`（シェア＝無料支援）
3. **フッター** — noteリンク常設
4. **プレリリースバナー** — 「Xフォロー＝応援」明示

### コピー原則

| NG | OK |
|----|-----|
| 寄付してください | この整理、続けたい？ |
| 有料会員登録 | 週次で追いかける（¥500/月） |
| 応援よろしく | 役に立ったら X で紹介 or note メンバー |

### 心理トリガー

- **公平性** — 「党の宣伝ではない。出典付き整理」
- **時計** — 「公言からNカ月、変わっていない事実を追うコスト」
- **社会証明** — Xシェア数・😊数（将来）
- **低摩擦** — Xシェア1タップ / noteは外部（特商法はプラットフォーム側）

## A/Bテスト設計（PDCA）

`src/lib/site-config.mjs` の `supportAbVariant: "a" | "b"` で切替。  
**2週間×1指標** で回す（同時多変量はしない）。

### テスト1: 支援CTAコピー（記事末尾）

| | A（感情） | B（価値） |
|---|-----------|-----------|
| 見出し | この整理、続けたい？ | 追いかけ続けるには |
| 本文 | 役に立ったら note メンバーで | 週次ダイジェスト＋深掘り |
| 計測 | `utm_campaign=ab_a` | `utm_campaign=ab_b` |

**勝ち指標:** note リンク CTR（Cloudflare Web Analytics カスタムイベント or UTM）

### テスト2: シェア導線位置（4週目〜）

| | A | B |
|---|---|---|
| ShareBar | タイトル直下 | 記事末尾（Comments前） |
| 勝ち指標 | `data-track=share-x` クリック / PV |

### テスト3: 支援ボタン文言（6週目〜）

| | A | B |
|---|---|---|
| CTA | note メンバーになる | 週次ダイジェストを受け取る（¥500） |
| 勝ち指標 | note 遷移数 |

### テスト4: Xフォロー可視性（8週目〜）

| | A | B |
|---|---|---|
| ヘッダー | テキストリンクのみ | ロゴ横に X アイコン |
| 勝ち指標 | @seiji1192site フォロー増（X Analytics） |
