# 公言と行動の整理表（政策 × 政党マトリクス）

最終更新: 2026-06-27  
**オーナーGO:** 2026-06-26

---

## 一言

**事実を見やすく整理するだけ。** 嘘かどうかは判定しない。「言ったこと」と「その後の行動」を並べ、ユーザーが自分で見る。

**公約不履行を「当たり前」として書かない。** 検討中・慎重論だけで、約束と経過日数を消す要約は禁止（`docs/writer-editorial.md`）。

---

## やらないこと / やること

| × | ○ |
|---|---|
| 嘘つき判定・信頼性スコア | 公言（出典付き）と行動（投票・法案）の対比 |
| 個人の品格評価 | 争点ごとの ◎▲❌？（定義に基づく整理） |
| 選挙向けの投票誘導 | 追跡中件数の事実カウント |

## UI イメージ

```
              物価対策   消費税   防衛費   …
自民           ▲         ▲       ◎
立憲           ◎         ❌      ▲
国民民主       …
```

各セルは **2行**:

| 行 | 内容 |
|----|------|
| **方針** | 公約・党首発言・政策文書（引用＋URL） |
| **行動** | 採決・提出法案・閣議決定（議案ID・speechURL） |

セル下に **記号**（行動行に対する整理）:

| 記号 | 定義 |
|------|------|
| ◎ | 公言の**方向**と行動が一致し、規模もおおむね一致 |
| ▲ | 方向は一致。**規模・期限・対象**が一部ずれ（例: 0%→5%引き下げ） |
| ❌ | 方向が逆、または公言と明確に矛盾する行動 |
| ？ | 行動データなし・解釈に争い・未判定 |

### 判定例（消費税）

| 公言 | 行動 | 記号 |
|------|------|------|
| 消費税0% | 5%引き下げに賛成 | **▲** |
| 消費税0% | 増税法案に賛成 | **❌** |
| 消費税0% | 関連法案未提出 | **？** |

## 公約優先ルール（2026-06-27 追加）

### 原則
公約は国民との約束であり、国会発言より評価軸として上位に置く。

### ルール

1. **公約がある場合は公約を評価軸の最上位とする**
   - 選挙公報・党マニフェスト・党HPの政策文書を一次ソースとして使用
   - `policyPledgeUrl` に公約ソースのURLを必ず記録する
   - 出典なし公約は使用しない

2. **symbolReason に「公約との具体的ギャップ」を一文記載する**
   - 例: 「公約『ゼロベース見直し』に対し、国会での実績は上限検討の表明のみ」
   - 国会発言だけを根拠に◎とするのは公約との乖離を無視することになるため禁止

3. **公約がない場合は「公約なし・発言ベース評価」と明示する**
   - `policyPledgeUrl` が空の場合、UIおよび symbolReason に「公約なし」を表示
   - 読者が「公約ベース評価済み」と誤解しないようにする

### データモデルへの追加フィールド

```json
{
  "partyLabel": "自由民主党",
  "policyPledgeUrl": "https://…（公約ページURL、なければ空文字）",
  "policyPledgeSummary": "外国人政策をゼロベースで見直す（2025年選挙公約）",
  "stance": { "…": "…" },
  "action": { "…": "…" },
  "symbol": "△",
  "symbolReason": "公約『ゼロベース見直し』に対し、国会実績は上限検討の表明のみ（乖離あり）"
}
```

## アルゴリズム（3段階）

### Step 1 — 事実抽出（自動）

| 種別 | ソース |
|------|--------|
| 方針 | 政権公約・党政策集・マニフェストPDF・党首発言（URL保存） |
| 行動 | 国会議事録API・採決記録・法案番号・予算案賛否 |

### Step 2 — 方向分類（ルール＋AI下書き）

政策軸ごとに行動を分類:

- `強める` / `弱める` / `維持` / `逆`

AIは **下書きのみ**。最終記号は Step 3 のルールエンジン。

### Step 3 — 記号付与（固定ルール）

**評価軸の優先順位:**
1. `policyPledgeUrl` あり → **公約を軸**に行動を評価
2. `policyPledgeUrl` なし → 国会発言・党方針を軸に評価（「公約なし」を明示）

```
if 公約あり:
  if 公約の方向と行動が一致 && 規模おおむね一致 → ◎
  if 公約の方向と行動が一致 && 規模ずれ        → ▲
  if 公約と行動が乖離・未着手                  → △（公約ギャップ）
  if 公約と行動が逆・明確矛盾                  → ❌

if 公約なし:
  if 発言の方向と行動が一致 && 規模おおむね一致 → ◎
  if 発言の方向と行動が一致 && 規模ずれ        → ▲
  if 発言と行動が逆 or 明確矛盾               → ❌
  if データ不足                              → ？
```

**ハルシネ禁止:** 原文・議案にない数字・投票を書かない。

## 政治家ページとの関係

- **党マトリクス** = 党の公式公約・党議決・代表発言ベース
- **個人ページ** = その議員の発言・個人投票（`docs/politician-pages.md`）
- 個人に **信頼性点数は付けない**。争点ごとの ◎▲❌ と **件数サマリ** のみ

件数サマリ例:

> 追跡中 12 件 — 一致 ◎8 / 部分 ▲3 / 不一致 ❌0 / 未判定 ？1

## 抜粋の選び方（政党・議員）

**全件掲載ではない。** 案件ページではその争点に関係する主体だけを載せる。

### 政党

| 優先 | 条件 |
|------|------|
| 1 | 当該争点の **国会審議**（採決・質疑）で党として立場が確認できる |
| 2 | **党公約・政策文書**に争点が明記されている |
| 3 | 衆参で議席があり、当該争点の審議に **実際に関与** している |

PoC・初期はライターが **手動選定**（例: 物価 → 自民・立憲・国民民主・維新）。将来は `data/policy-matrix/{slug}.json` の `parties[]` に理由付きで登録。

`excerpt.parties` にページ上の説明文を書く（ユーザー向け）。

### 議員

| 優先 | 条件 |
|------|------|
| 1 | **本案件の primarySpeech**（メイン国会発言）の議員 |
| 2 | 同一争点で **与野党が対比できる** 別発言者（国会URLあり） |
| 3 | 閣僚・党首など **争点の実行責任者** |

`stanceMatrix.highlightPoliticians` に slug 列挙。全議員ページへの代替ではない。

`excerpt.politicians` にページ上の説明文を書く。

### タイトル

案件ページの見出しは **`{政策名}に対する公言と行動`**（例: 物価高対策に対する公言と行動）。`policyLabel` または記事タイトルから自動生成。
