最終更新: 2026-06-26  
前提: サンプル UI v1 反映後。オーナー「他にユーザーが必要としそうな提案は？」への回答。

---

## 優先度 A（公開前〜直後に効く）

| # | 提案 | ユーザー価値 | コスト感 |
|---|------|-------------|----------|
| 1 | **案件更新通知（RSS + メール任意）** | 「あの話」追いかけ層の再訪。Push 不要でも RSS は技術者・ニュース好きに刺さる | 小（RSS は静的生成、メールは Resend 等） |
| 2 | **用語辞典の横断検索** | 記事を読まなくても「補助金」「リコール要件」を即理解。SEO ロングテール | 中（D1 FTS + 辞典ページ 1 枚） |
| 3 | **削除 X の Before/After 比較 UI** | 差分が一目で分かり「何が消されたか」が伝わる。サイトの差別化核 | 中（画像 2 枚 + スライダー CSS） |
| 4 | **モバイル 😊😠 の親指ゾーン固定** | 案件読了後の投票率向上。フロートボタンは既存、**初回タップ説明**を追加 | 小（CSS + 1 回限りツールチップ） |
| 5 | **「いまの結論」ボックスの共有用 OGP** | X 引用時に結論サマリが画像化され拡散しやすい | 中（Workers + @vercel/og 相当） |

## 今回 UI で反映済み

- ヘッダーロゴ拡大（横長 PNG 4rem / モバイルはマーク 3.25rem）
- 検索窓はヘッダー 1 つのみ（hero 内削除）
- 一覧カテゴリチップ + ソート説明（関心順 vs 話題順）

## CEO 判断を仰ぎたい 1 点

**通知:** RSS のみ先行 vs メール登録も MVP に入れるか。  
再訪率 KPI が最優先なら **RSS + X 自動投稿** がコスパ最高。
