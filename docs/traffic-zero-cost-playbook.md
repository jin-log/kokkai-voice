# 0円集客・インデックス・SEO プレイブック

最終更新: 2026-06-28  
対象: 日本の政治なう（https://seiji1192.site）プレリリース期  
方針: **費用0円・無料枠のみ**。有料広告・有料PRは別フェーズ。

関連: `docs/marketing-report-ceo.md` · `docs/sns-strategy.md` · `docs/pre-release.md`

## 使い方

1. **初週** — 「A. インデックス」「B. 画像PNG」「C. 無料PR」から ★ だけ実行
2. **毎週** — 案件追加のたび「D. SNS」「E. 被リンク」「F. コンテンツ再利用」
3. **月1** — GSC でインデックス率・クエリ確認 → 足りないチャネルを追加

## A. 検索エンジンに「とにかく載せる」（0円）

| # | 手法 | 具体 | SEO効果 | 担当 |
|---|------|------|---------|------|
| A1 | **Google Search Console** | プロパティ追加 → `sitemap-index.xml` 送信 → URL検査で主要案件を「インデックス登録をリクエスト」 | ★★★ | オーナー/CEO |
| A2 | **Bing Webmaster Tools** | GSC からインポート可。サイトマップ同送 | ★★ | CEO |
| A3 | **IndexNow** | Bing/Yandex 向け即時通知。CF Pages Functions または手動 ping（主要URL更新時） | ★★ | CEO |
| A4 | **Google ビジネスプロフィール** | 「オンラインのみのサービス」として登録（実店舗不要）。サイトURL・カテゴリ「ニュースメディア」 | ★ | オーナー |
| A5 | **robots.txt / sitemap 確認** | `/dev/` 除外済みか・案件URLが sitemap に載るか定期確認 | ★★★ | CEO |
| A6 | **canonical 統一** | `https://seiji1192.site/case/{slug}/` 末尾スラッシュ固定。重複URLなし | ★★★ | CEO |
| A7 | **構造化データ** | WebSite + Article JSON-LD（C26 タスク）。リッチリザルト・Discover 向け | ★★★ | CEO |
| A8 | **meta description 全ページ** | 案件名＋「あの話どうなった？」＋平易語1行（120字前後） | ★★★ | CEO |
| A9 | **内部リンク網** | トップ→案件、案件→関連案件3本、用語→案件。クローラ巡回用 | ★★★ | CEO |
| A10 | **404・リダイレクト監査** | リンク切れ＝クロール浪費。デバッガー B1/B4 | ★★ | CEO |

## B. 画像・PNG で拡散・インデックス（0円）

**思想:** テキストだけより「1枚図」はシェア・保存・画像検索に強い。

| # | 手法 | PNGの中身 | 投稿先 | SEO/流入 |
|---|------|-----------|--------|----------|
| B1 | **OGP 案件別** | タイトル＋ひよこ＋「出典付き」ロゴ。1200×630 | 各案件の `og:image`（X/LINE/Slack シェア時自動） | ★★★ |
| B2 | **X 投稿用インフォ1枚** | 「公言 vs その後」3行＋QR/短URL | X 1枚目画像付きツイ | ★★ |
| B3 | **X スレッド各帖PNG** | 1/7=サイト全体、2/7=仕組み、3/7=案件スクショ… | `docs/pr/pre-release-announce.md` スレッドとセット | ★★ |
| B4 | **案件サマリカード** | Canva無料：見出し＋3 bullet＋URL（ブランド色） | X / note アイキャッチ / はてブ OGP 代わり | ★★ |
| B5 | **タイムライン縦長PNG** | 日付→出来事→出典（スクロール1枚） | X・Threads・note 本文内 | ★ |
| B6 | **「約束から○日」カウンター画像** | 数字大きく＋案件名 | 更新のたび X | ★★ |
| B7 | **Pinterest** | 案件カード・タイムライン縦長。各ピンに案件URL | pinterest.com（無料アカ） | ★★ 画像検索・長期 |
| B8 | **note / はてな サムネ** | 1280×670 前後。タイトル文字大きく | 各プラットフォームのアイキャッチ | ★★ |
| B9 | **GitHub Social Preview** | リポ public 化する場合 `docs/assets/og-default.png` | GitHub リポ（構築記事とセット） | ★ |
| B10 | **Wikipedia  Commons 非推奨** | 自サイト宣伝目的の画像投稿は方針外。**国会出典の引用図**のみ検討 | — | △ |
| B11 | **Imgur / Gyazo** | 長文説明用。説明文に **必ず本家URL** | Reddit pitch・フォーラム貼付 | ★ |
| B12 | **LINE OpenChat カバー** | 政治リテラシー系OC作る場合のカバー＋固定メッセにURL | LINE | ★ |
| B13 | **PDF 1枚チラシ** | A4→PDF→Google Drive **リンク公開**（インデックスされる） | pitch メール添付・PR本文 | ★★ |
| B14 | **スクリーンショット「Before/After」** | X削除前後（サイト既存機能）。「証拠付き」差別化 | X・PR本文・note | ★★★ |

**PNG制作ツール（無料）:** Canva / Figma / Photopea / スマホ「文字入れ」アプリ

## E. 被リンク・引用（0円・SEO核心）

| # | 手法 | やり方 |
|---|------|--------|
| E1 | **無料PR 3〜5本** | 本文に `https://seiji1192.site` 直リンク |
| E2 | **Wikipedia 編集** | 該当案件の「脚注」に**国会議事録・報道と並べて**本サイトを「解説」として追加（宣伝文禁止） |
| E3 | **Qiita / Zenn 構築記** | 「国会API×Astro」技術記事末尾にサービスURL |
| E4 | **GitHub README** | リポ public 時 badges + デモURL |
| E5 | **市民団体・NPO** | 資料PDFの「参考リンク」に掲載依頼（メール1通） |
| E6 | **大学ゼミ・シラバス** | 政治リテラシー授業の「参考サイト」リスト依頼 |
| E7 | **Podcast ショーノート** | 出演 or リクエスト → ショーノートにURL |
| E8 | **相互リンク** | jin-log 等**別プロジェクト**から1行リンク（内容被り注意） |
| E9 | **HARO / PRONE 返答** | 記者質問に「国会とSNS乖離の整理例」としてURL提示 |
| E10 | **プレスキット1URL** | Notion/Google Doc：ロゴPNG・スクショ3・代表案件3・一言pitch |

## G. キーワード・ロングテール（0円 SEO）

狙うクエリ例（案件ページ title/H1/description に自然に含める）:

- `{案件名} どうなった`
- `{案件名} 最新`
- `{案件名} 国会 発言`
- `{案件名} 経過`
- `{政治家名} {テーマ} 発言`（個人ページ将来）

**やらない:** キーワードスタッフィング・自動生成数千ページ・Doorway

## J. やらない・後回し（0円でもコスト高）

| 項目 | 理由 |
|------|------|
| 有料 PR TIMES | 初月不要（`marketing-report-ceo.md`） |
| 5ch 自作まとめ | 炎上・BAN リスク |
| Wikipedia 宣伝編集 | 削除・信用失墜 |
| キーワード自動量産 | Google スパム判定 |
| 政治広告（X Ads 等） | 費用＋公選法リスク |
| 過剰 DM 宣伝 | ブロック・評判低下 |
