# 公開ゲート vs 100%完成

最終更新: 2026-06-27

---

## 用語

| 用語 | 意味 |
|------|------|
| **公開ゲート** | `check-case-page` 合格 = ①〜④が全部OK。これ以外は **ビルドに載らない** |
| **100%完成** | 公開ゲート + ⑤デプロイ + ⑥デバッグOK |

---

## 順番（これ以外は漏れる）

```
①コンテンツ → ②〇× → ③X検証 → ④法務 → ⑤デプロイ → ⑥デバッグ
```

| 段 | 誰 | タイミング |
|----|-----|-----------|
| ①〜③ | ライター | デプロイ前 |
| ④ | 法務 | **デプロイ直前**（内容が揃ってから） |
| ⑤ | CEO | `npm run deploy` |
| ⑥ | デバッガー | **本番URLを見てから** |

**法務は公開の後ではない。** ④がOKになるまで⑤に進めない（機械ゲート）。

**デバッグは公開の前ではない。** 本番でリンク・表示を確認してから100%にする。

---

## 以前のミス（再発防止）

| 問題 | 原因 | 対策 |
|------|------|------|
| X未了なのに防衛費が本番に出た | H1が「X枠なし＝OK」だった | **X検証2件必須**に変更 |
| 経緯・〇×が別トピック | 発言冒頭切り取り／長い答弁を丸採点 | **話題ウィンドウ抜粋** + B4/D2/E4/G6 |
| いまの結論が逐語抜粋 | 機械要約のみ | **結論3行型** + B4（重複・途中切れ禁止） |
| 完成度%で法務と公開が並列に見えた | チェックリストUIが順序を示さなかった | ①〜⑥パイプライン表示 |
| 93%なのにデバッグ未実施 | デバッグが項目に無かった | **⑥デバッグ**を100%に追加 |

```powershell
node scripts/check-case-page.mjs --slug bouka-taisaku  # 公開ゲート

```
