## オーナー方針（Skill内固定・verbatim）

**公言して終わりは記載しない。** 公言の日付・経過時間・その間に変わった事実（変わっていなければ「変更なし」）・公式説明を並べ、読者に判断させる。

**公言してるから良し、が一番ダメ。** 政治家の言い訳を代弁しないが、国会・会見・報道で述べられた**公式理由**は事実として載せる。

## 5スキル（教育順）

### 1. 時計スキル

- 起点は3つから選ぶ（複数可）：**公約日** / **政権発足・就任日** / **本人が言った期限**
- nowSummary 1行目：**未実施・一部実施・実施済み** ＋ **経過日数・月数**
- 禁止：「前向きに検討中」だけで終わる1行

### 2. 公約優先スキル

- 国会の最新発言で公約を上書きしない
- 公約URLまたは国会での公約言及がある党に「公約なし」と書かない
- ◎▲❌ は**行動**に付ける。公言だけでは ◎ にしない

### 3. 停滞の事実スキル（核心）

読者が知りたい4列を常に並べる：

| 列 | 内容 |
|----|------|
| 公言 | 誰が・いつ・何と言ったか（出典URL） |
| 期限 | 「早期」「夏までに」等 |
| 経過 | 起点から何日・何カ月 |
| その後 | 法案・予算・訴訟進行等。**変わっていなければ「変更なし」** |

**書いてよい「なぜ進んでいないか」**

- 公式説明（「国民会議の結論待ち」等）＋出典
- 客観的事実（法案未提出、予算未計上、閣議未決定）
- 報道（「〜と報じられている」＋URL）

**書いてはいけない**

- 「言い訳だ」「本音は〜」
- 進捗なしなのに「努力している」で締める
- 検討中だけで 1〜3 列を省略

**arcSummary / nowSummary 2行目の型**

```
{日付} {出来事（事実）}
→ {今日} 時点、{変化した事実 or 変更なし}（約{N}カ月経過）
→ {当事者}は「{公式説明の要約}」（出典）
→ {第三者：検察・選管等}は{コメント有無の事実}
```

### 3b. タイムライン・メリデメ（2026-06-28）

**タイムライン:** X3 + 国会3 = 計6以上（`enrich-timeline-all.mjs`）  
**prosCons:** メリ2・デメ2、各 `figure` + `sourceUrl`（経緯の上に表示）

**出典等級（2026-07-18）:** 公式・国会を正本。Xは補助（`extraSources`）。X単独で「対策は動いている」等は不可。  
**要約禁止:** 「公開されている報道・試算を時系列で…」／日付＋議事断片の羅列。  
**slugズレ:** 芯戻し or 改名＋301。タイトルだけ変えて放置しない。  
正本: `docs/writer-editorial.md`

### 3c. セクション別ライター規則（2026-06-30・オーナー追記）

| セクション | 役割 | 必須 | 禁止 |
|-----------|------|------|------|
| **経緯** `arcSummary` | 日付ごとの出来事 | 日付が違えば文も違う | 同文のコピペ |
| **根拠** `summaryBullets` | 結論の裏付け（数値・主体・法案） | 経緯と別角度 | テンプレ「国会論点」連打 |
| **メリデメ** | 制度の得失 | 読者メリット・デメリット | API件数・言及件数 |
| **〇× 方針** | 会派の立場要約（1〜2文） | 第三者目線「〜立場。」 | 議事録抜粋 |
| **〇× 行動** | 提出・可決・質疑の**中身** | 動詞＋対象 | 「〇〇での発言」だけ／**方針と同一引用** |

**絶対禁止:** 方針＝原文切り出し、行動＝「日付＋会議＋同じ原文」。法務 L9 / `stanceActionDistinct` で落ちる。

実装: `scripts/lib/writer-synthesize.mjs`（`synthesizeArcSummary` / `synthesizeEvidence` / `synthesizePartyMatrix` / `synthesizeProsCons`）— **全国会記事で `complete-article.mjs` が自動適用**

### 3d. 効果検証型（2026-07-19・能登復興予算で確立）

**対象:** 予算・復興・交付金・予備費など「いくら出して何が起きたか」が主題の案件。  
**正本サンプル:** `data/articles/noto-fukko-budget.json`

読者の疑問に答える4軸を**数字の突き合わせ**で並べる（評価断定しない）:

| 軸 | 書くこと | 未公表のとき |
|----|----------|--------------|
| **内訳** | 何にいくら（インフラ／なりわい／住宅等） | — |
| **必要額との差** | 被害試算・必要額 vs 予算規模（比率可） | 「公式の必要額試算は見当たらない」と明記 |
| **進捗** | 仮設入居・本復旧・恒久住宅・人口流出など現地到達 | 「一本集計なし」も事実として書く |
| **不正の有無** | 会計検査院・監査の有無と結果 | 「摘発は本稿時点で未確認」 |

**ストック被害**（建物・インフラ等の物的損失試算）と**フロー予算**（予備費・補正・基金）を混同しない。両方載せ、比較の前提を1文で説明。

**禁止**

- 「足りない／十分」など評価断定（数字と出典で読者に判断させる）
- 予算総額だけ載せて内訳・進捗を省略
- 公式にない「不足あと〇億円」を報道単価から逆算して断定

**タイトル:** `kokkai-headlines` の型6（検証軸並べ）。

### 4. ギャップ言語化（policy-matrix）

- `symbolReason` は **「公言A に対し、行動B（日付）」** 1文必須
- ▲＝方向近いが**期限・規模ずれ**
- ❌＝**逆方向** or 明確矛盾
- ？＝行動データ不足（無理に◎にしない）

### 5. 平易語スキル

- `plainExplanation`＝時系列で追える短文
- `glossary` に専門用語 2〜4 語
- 数字・日付は公約PDF・国会発言・報道URLのみ。**推測日禁止**

## nowSummary 3行の型

1. **公約 vs 現状（時計）** — 誰が何と約束／いまの状態／公約から何カ月
2. **言った期限 vs 経過** — 「早期」「〇月までに」と期限から何日・何カ月
3. **いまの公式説明（中立）** — 「検討中」も事実だが、1・2を省略しない

## 案件タイプ別「行動」の定義

| タイプ | 行動に数えるもの |
|--------|------------------|
| 国会 | 法案・予算・採決・閣議・議事録URL |
| 地方 | 条例・知事会見・選管届・訴訟進行（公表分） |
| 選挙 | 告示・公報記載・当落 |
| 行政 | 省庁発表・ガイドライン・人事（出典付き） |

### 6. 読者の疑問スキル（タイトル・冒頭）

**詳細は `kokkai-headlines` スキル。** ここは要約のみ。

**国民がクリックするのは「政策名」ではなく「自分の具体的な疑問」。**

| × 抽象（クリックされない） | ○ 具体（疑問が解ける） |
|---------------------------|------------------------|
| 物価高対策 | 物価高で確保した1兆円予備費、何に使われた？ |
| 食料品の消費税 | 食料品消費税ゼロ公約、スーパーはまだ8%？ |
| 防衛費・安保 | 防衛費増で増税はある？安保三文書と財源 |
| 出生率改善（政策名だけ） | 出生率改善予算とその成果実績 |

**タイトルの型（末尾 `— あの話どうなった？` は付けない。案件名がそのまま H1）**

1. **数字＋結果** — `3.6兆円の少子化予算、出生率は上がった？`
2. **とは？** — `副首都構想って何？` / `太陽光パネル設置義務とは？`
3. **メリデメ・推移** — `大阪都構想のメリットデメリット` / `不法移民の国内数推移`
4. **公約vs現状** — `消費税ゼロ公約、スーパーはまだ8%？`
5. **いつ・どうなった** — `告発から2年、結局どうなった？`

**内容の心理（中立のまま）**

- 読後感は **「今そうなってるのね」「そういう事だったのか」**
- `plainExplanation` 1段落目＝**タイトルの疑問に直接答える**（定義→数字→いま）
- `nowSummary` 1行目＝タイトルで開いた疑問の**結論先出し**
- 党の応援・国民感情の代弁はしない。**事実で疑問を解く**

**禁止**

- `{政策名} — あの話どうなった？` だけの汎用タイトル
- 見出しと中身がズレる（タイトルは消費税、中身は別テーマ）
- 煽り評価語（裏切り・見送り・嘘つき）をタイトルに

サイトキャッチ「あの話どうなった？」は**ブランド**。**案件タイトルには重ねない。**
