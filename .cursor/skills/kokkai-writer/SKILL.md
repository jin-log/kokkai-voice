# KV ライター Skill

## いつ使う

- `data/articles/*.json` の作成・更新
- `nowSummary` / `arcSummary` / `plainExplanation` / `glossary`
- `data/policy-matrix/*.json` の ◎▲❌ と `symbolReason`
- 記事完成前の自己チェック

## 必読（正本）

1. `docs/writer-editorial.md` — 編集方針
2. `docs/policy-matrix.md` — 〇×・公約優先
3. `.cursor/rules/仕事/kokkai-voice/writer.mdc` — データ規約・API

## オーナー方針（Skill内固定・verbatim）

**公言して終わりは記載しない。** 公言の日付・経過時間・その間に変わった事実（変わっていなければ「変更なし」）・公式説明を並べ、読者に判断させる。

**公言してるから良し、が一番ダメ。** 政治家の言い訳を代弁しないが、国会・会見・報道で述べられた**公式理由**は事実として載せる。

## 5スキル（教育順）

### 1. 時計スキル

- 起点は3つから選ぶ（複数可）：**公約日** / **政権発足・就任日** / **本人が言った期限**
- nowSummary 1行目：**未実施・一部実施・実施済み** ＋ **経過日数・月数**
- 禁止：「前向きに検討中」だけで終わる1行

### 2. 公約優先スキル

- 国会の最新発言で公約を上書きしない
- 公約URLまたは国会での公約言及がある党に「公約なし」と書かない
- ◎▲❌ は**行動**に付ける。公言だけでは ◎ にしない

### 3. 停滞の事実スキル（核心）

読者が知りたい4列を常に並べる：

| 列 | 内容 |
|----|------|
| 公言 | 誰が・いつ・何と言ったか（出典URL） |
| 期限 | 「早期」「夏までに」等 |
| 経過 | 起点から何日・何カ月 |
| その後 | 法案・予算・訴訟進行等。**変わっていなければ「変更なし」** |

**書いてよい「なぜ進んでいないか」**

- 公式説明（「国民会議の結論待ち」等）＋出典
- 客観的事実（法案未提出、予算未計上、閣議未決定）
- 報道（「〜と報じられている」＋URL）

**書いてはいけない**

- 「言い訳だ」「本音は〜」
- 進捗なしなのに「努力している」で締める
- 検討中だけで 1〜3 列を省略

**arcSummary / nowSummary 2行目の型**

```
{日付} {出来事（事実）}
→ {今日} 時点、{変化した事実 or 変更なし}（約{N}カ月経過）
→ {当事者}は「{公式説明の要約}」（出典）
→ {第三者：検察・選管等}は{コメント有無の事実}
```

### 4. ギャップ言語化（policy-matrix）

- `symbolReason` は **「公言A に対し、行動B（日付）」** 1文必須
- ▲＝方向近いが**期限・規模ずれ**
- ❌＝**逆方向** or 明確矛盾
- ？＝行動データ不足（無理に◎にしない）

### 5. 平易語スキル

- `plainExplanation`＝時系列で追える短文
- `glossary` に専門用語 2〜4 語
- 数字・日付は公約PDF・国会発言・報道URLのみ。**推測日禁止**

---

## nowSummary 3行の型

1. **公約 vs 現状（時計）** — 誰が何と約束／いまの状態／公約から何カ月
2. **言った期限 vs 経過** — 「早期」「〇月までに」と期限から何日・何カ月
3. **いまの公式説明（中立）** — 「検討中」も事実だが、1・2を省略しない

---

## 案件タイプ別「行動」の定義

| タイプ | 行動に数えるもの |
|--------|------------------|
| 国会 | 法案・予算・採決・閣議・議事録URL |
| 地方 | 条例・知事会見・選管届・訴訟進行（公表分） |
| 選挙 | 告示・公報記載・当落 |
| 行政 | 省庁発表・ガイドライン・人事（出典付き） |

## 提出前チェックリスト

- [ ] nowSummary 1行目に **未実施 or 実施済み** がはっきりある
- [ ] **経過日数・月数** がある（起点日明示）
- [ ] 「検討中」だけで終わっていない
- [ ] 停滞している場合 **変更なし or 公式説明** がある
- [ ] policy-matrix に公約 or 国会言及の出典
- [ ] `symbolReason` にギャップ1文
- [ ] `node scripts/check-case-page.mjs --slug {slug}` が exit 0
